# ProsperPals Alpha Readiness Working Surface

This folder is the working surface for the current BMAD step:

**`alpha-readiness / interview-evidence-and-hosted-hardening-execution`**

The hosted-alpha **NO-GO remains locked** while this folder is populated.

## Purpose

Use this folder to convert the alpha decision packet into durable proof:
- interview logs,
- batch and final syntheses,
- hosted-hardening closure notes,
- receipt realism traces,
- operator-readiness evidence,
- and the inputs for the next formal GO / CONDITIONAL GO / NO-GO review.

## Rules

1. **Artifacts beat memory.** If it matters, write it down here.
2. **Thresholds beat vibes.** Use the thresholds already defined in the alpha-readiness docs.
3. **Proof beats status language.** If something is “done,” attach evidence or record the manual fallback.
4. **NO-GO stays locked.** This folder is for execution, not premature celebration.
5. **No wedge drift.** Product thesis changes require repeated evidence, not one-off opinions.

## Suggested operating rhythm

### Per interview
- update `interview-evidence-tracker.md`
- copy the session template into `evidence/interviews/`
- fill it the same day
- capture strongest quote and trust-drop moment

### Per batch of 3 interviews
- create one batch synthesis in `evidence/interviews/`
- mark the three sessions as `batched` in `interview-evidence-tracker.md`
- record any wording/UI change before the next batch

### Per hardening closure item
- add proof notes under the relevant evidence folder
- update the checklist status to complete / manual fallback / open blocker

### Before the next re-decision review
- update `redecision-dashboard-inputs.md`
- create a dated dashboard and memo in `evidence/redecision/`

## Current bar for success

This step is execution-credible only when the repo contains:
- 12 session logs,
- 4 batch syntheses,
- 1 final interview synthesis,
- one canonical interview tracker with cohort and threshold math,
- hosted-hardening evidence,
- receipt realism traces,
- operator/access readiness evidence,
- and a current re-decision dashboard input file.
