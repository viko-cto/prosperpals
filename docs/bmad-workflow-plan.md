# ProsperPals — BMAD Workflow Plan

## Goal
Create a production-ready ProsperPals planning set using BMAD v6.0 with zero shortcutting.

## Workflow Sessions

### Session 1 — Analyst: Product Brief
Command: `/bmad-bmm-create-product-brief`
Inputs:
- Problem: budgeting apps feel robotic, judgmental, and overwhelming
- Audience: Gen Z (18–28)
- Core value: emotional guidance + actionable financial behaviors
- Companions: Goldie (motivator), Fin (pragmatist)
- Business model: freemium + premium subscription

### Session 2 — PM: PRD
Command: `/bmad-bmm-create-prd`
Include:
- onboarding flows
- account sync strategy (open banking)
- goals, budgets, nudges, rituals
- retention loops and social mechanics
- trust/privacy and compliance requirements

### Session 3 — UX Design
Command: `/bmad-bmm-create-ux-design`
Include:
- design language (no purple/indigo)
- persona-driven tone
- low-friction daily check-in
- mobile-first interface

### Session 4 — Architecture
Command: `/bmad-bmm-create-architecture`
Initial stack hypothesis:
- Next.js app + Supabase + Vercel AI SDK
- secure tokenized bank-data integration
- event-driven recommendation engine

### Session 5 — Epics & Stories
Command: `/bmad-bmm-create-epics-and-stories`
Create P0/P1/P2 structure with strict acceptance criteria.

### Session 6 — Readiness Check
Command: `/bmad-bmm-check-implementation-readiness`
Must pass with no critical contradictions before coding.
